## UX Research

> ~615 tokens

### Research Planning

Always start with:
1. Define the research question
2. Choose method (survey, interview, usability test, feedback analysis)
3. Identify target audience
4. Estimate sample size needed for reliable findings
5. Define analysis plan before collecting data

### Survey Design Principles

- Keep surveys under 5 minutes (8-10 questions max)
- Start with screener questions to filter relevant respondents
- Use JTBD (Jobs-to-be-Done) framing for motivation questions
- Avoid leading questions and double-barreled questions
- Include both closed (quantitative) and open (qualitative) questions
- End with open-ended "anything else" question

### Question Types by Purpose

| Purpose | Question Type | Example |
|---------|--------------|---------|
| Screener | Multiple choice | "How often do you use X?" |
| Context | Multiple choice | "Where/when do you use X?" |
| JTBD | Select all that apply | "What are you trying to do when you open X?" |
| Pain points | Open text | "What is most frustrating about X?" |
| Importance | Likert scale | "How important is Y for your continued use?" |
| Preference | Multiple choice | "Which would you prefer: A or B?" |
| Priority | Drag to rank | "Rank these features by importance" |

### NPS Feedback Analysis Framework

```
NPS Score: +<N> (<percent>% promoters - <percent>% detractors)

Promoter Themes (what they love):
| Theme | Mentions | Example Quote |

Detractor Themes (what is broken):
| Theme | Mentions | Severity | Example Quote |

Passive Themes (what would convert them):
| Theme | Mentions | Example Quote |

Key Insight: <the #1 actionable finding>
```

### Thematic Analysis Workflow

1. Read all responses to get overall sense
2. Code responses with tags (pain point, feature request, praise, etc.)
3. Group codes into themes
4. Count frequency of each theme
5. Rank by frequency x business impact
6. Present themes with representative quotes

### Feature Request Prioritization

| Factor | Weight |
|--------|--------|
| Frequency of request | 30% |
| Revenue impact (which segments request it) | 30% |
| Implementation effort | 20% |
| Strategic alignment | 20% |

### Rules

- Always specify the research question before designing a study
- Distinguish between what users say they want and what behavior data shows
- Include sample sizes and confidence levels for quantitative findings
- Recommend further research when findings are inconclusive
- Connect findings to actionable product decisions
