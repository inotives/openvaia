## Feedback Analysis

> ~950 tokens

### Collection Strategy

Gather feedback across five channel types:
- **Proactive**: in-app surveys, email campaigns, user interviews, beta feedback programs
- **Reactive**: support tickets, app store reviews, social media mentions, community forums
- **Passive**: usage analytics, session recordings, heatmaps, behavioral patterns
- **Community**: forums, Discord, Reddit, user groups, developer communities
- **Competitive**: review sites, social media, industry forums, analyst reports

### Processing Pipeline

1. **Ingest** — collect from all sources via APIs, standardize format
2. **Clean** — deduplicate, normalize text, validate quality, assign quality score
3. **Sentiment** — automated emotion detection, satisfaction scoring, confidence level
4. **Categorize** — tag themes, assign priority (P0-P3), classify impact (high/med/low)
5. **QA** — manual review sample, accuracy validation, bias check, stakeholder review

### Quantitative Analysis

- **Volume analysis**: feedback frequency by theme, source, and time period
- **Trend analysis**: pattern changes over time with seasonality detection
- **Correlation**: feedback themes vs. business metrics (NPS, churn, revenue) with significance testing
- **Segmentation**: differences by user type, geography, platform, cohort
- **Satisfaction modeling**: NPS, CSAT, CES score correlation with predictive modeling

### Qualitative Synthesis

- **Verbatim compilation**: representative quotes by theme with preserved context
- **Story development**: user journey narratives with pain points and emotional mapping
- **Edge cases**: uncommon but critical feedback with impact assessment
- **Emotional mapping**: frustration and delight points with intensity scoring
- **Context factors**: environmental conditions affecting feedback

### Prioritization Frameworks

Apply one or more depending on context:
- **RICE**: Reach x Impact x Confidence / Effort
- **MoSCoW**: Must have / Should have / Could have / Won't have
- **Kano model**: Basic (must-be) / Performance (one-dimensional) / Excitement (attractive)

### Delivery Formats

**Executive Dashboard:**
- Real-time sentiment and volume trends with alerts
- Top priority themes with business impact estimates and confidence intervals
- Satisfaction KPIs with benchmarks and competitive comparison
- ROI tracking for feedback-driven improvements

**Product Team Report:**
- Feature request analysis with user stories and acceptance criteria
- User journey pain points with improvement recommendations and effort estimates
- A/B test hypotheses generated from feedback themes with success criteria
- Development priority recommendations with supporting data

**Customer Success Playbook:**
- Common issue resolution guides with response templates
- Proactive outreach triggers for at-risk segments
- Education content suggestions based on confusion points
- Success metrics for feedback-driven improvements

### Continuous Improvement

- Track theme accuracy (target: 90%+ stakeholder validation)
- Measure actionability (target: 85% of insights lead to decisions)
- Monitor prediction accuracy for feedback-driven feature success
- Optimize collection channels based on response quality and volume
- Reduce bias through methodology refinement and diverse sourcing
